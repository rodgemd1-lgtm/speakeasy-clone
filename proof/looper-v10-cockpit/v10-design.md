## V10 Product Promise

`speakeasy-clone` is a design-language and asset archive for a client barbershop
site (`speakeasybarberlounge.com`). Its V10 state is one of:

- **A**: A clean archived bundle with a ProofPacket, consolidated into
  `speakeasy-barbershop` and marked `archived` in the estate map.
- **B**: A promoted canonical design-system component living in
  `speakeasy-barbershop` with this repo redirecting to it.

The consolidation action on record is `archive-after-asset-harvest`.

---

## 10 Animation Lenses (Concise)

1. **Future-back product**: archived client asset with auditable ProofPacket.
2. **User workflow**: clone → `make smoke` → review design-language → harvest → ProofPacket → archive.
3. **Agent workflow**: planner reads V10 issue → runner runs smoke → reviewer checks proof → human approves archive.
4. **CLI**: `speakeasy-clone {info,capabilities,smoke,doctor,clone}` — all local, no secrets.
5. **MCP**: read-only tools for design-language inspection, smoke, and repo info. No mutations.
6. **Data**: design tokens in `.designlang/`, proof under `proof/`, no private exports in repo.
7. **Deterministic layer**: `make smoke` exits 0 locally before any PR merges; no network required.
8. **Design system**: `.designlang/` contains full extracted tokens, components, and voice for reuse.
9. **Quality**: smoke gate + shell `bash -n` check + JSON parse check + human review before archive.
10. **Weekly improvement**: `weekly-smoke.yml` runs Monday 06:00 UTC, writes `smoke-latest.txt`, no auto-merge.

## Quality Gates

1. `make smoke` exits 0 (checks key files + CLI self-check).
2. All shell scripts pass `bash -n` (syntax-only check).
3. All JSON files parse: `python3 -m json.tool <file>`.
4. No secrets committed: manual review for `.env`, token, credential patterns.
5. Human approval before: archive, consolidation merge, any destructive operation.
